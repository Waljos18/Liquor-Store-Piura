2.2 Nombre del Proyecto
Sistema Web y App Móvil con IA para Gestión Integral de Licorería

2.3 Descripción
Desarrollo e implementación de un sistema web y aplicación móvil con inteligencia artificial que permita a las licorerías pequeñas en Piura (1-2 empleados) gestionar de manera integral y automatizada sus operaciones de venta, inventario, facturación electrónica (SUNAT), promociones y reportes. El sistema incluye módulos de punto de venta (POS), control de inventario en tiempo real, emisión de comprobantes electrónicos, gestión de promociones y packs, programa de fidelización, reportes analíticos y funcionalidades de inteligencia artificial para recomendaciones de productos, predicción de demanda y optimización automática de inventario.

2.4 Dirección / Áreas involucrados
- Gerencia/Administración (Dueño de la Licorería)
- Área de Ventas/Caja (Punto de Venta)
- Área de Almacén/Inventario (Control de Stock)
- Área de Contabilidad/Finanzas (Facturación Electrónica SUNAT)

2.5 Objetivos Estratégicos

Estratégico
Descripción
1
Digitalización de Operaciones
Digitalizar completamente el proceso de ventas mediante sistema POS rápido e intuitivo, eliminando procesos manuales propensos a errores y reduciendo el tiempo de venta de 3-5 minutos a 30-45 segundos.
2
Cumplimiento Normativo
Automatizar la emisión de comprobantes electrónicos (boletas y facturas) integrados con SUNAT, garantizando el 100% de cumplimiento normativo y eliminando multas por incumplimiento.

Optimización con IA
Implementar inteligencia artificial para automatizar decisiones de negocio (recomendaciones de productos, predicción de demanda, optimización de inventario), permitiendo que negocios pequeños compitan con herramientas avanzadas normalmente disponibles solo para grandes empresas.

2.7 Supuestos y premisas
Supuestos Descripción
1
Conectividad a Internet
Se asume que la licorería cuenta con conexión a internet estable para el funcionamiento del sistema web y sincronización de datos. El POS tendrá funcionalidad offline básica para operar sin conexión temporal.
2
Certificado Digital SUNAT
Se asume que el cliente cuenta o puede obtener certificado digital y cuenta con OSE (Operador de Servicios Electrónicos) configurado para la emisión de comprobantes electrónicos.
3
Disponibilidad del Cliente
Se asume que el dueño/empleado tiene disponibilidad para participar en reuniones de requerimientos, pruebas y capacitación durante el desarrollo del proyecto (14 semanas).

Premisa Descripción
1
Capacidad Tecnológica del Usuario
El dueño/empleado tiene conocimientos básicos de tecnología y puede usar sistemas informáticos con capacitación adecuada. El sistema está diseñado para ser intuitivo y requerir mínima capacitación.
2
Estabilidad de Requerimientos
Los requerimientos principales del sistema (ventas, inventario, facturación, promociones) son estables y no cambiarán significativamente durante el desarrollo. Cambios menores se gestionarán mediante metodología ágil.
3
Datos Históricos para IA
Se asume que durante los primeros meses de operación se generarán suficientes datos de ventas para entrenar y mejorar los modelos de IA. Los modelos iniciales funcionarán con datos de prueba y mejorarán con el tiempo.

2.8 Restricciones
Restricción Descripción
1
Tiempo de Desarrollo y Recursos Humanos
El proyecto debe completarse en 14 semanas (3.5 meses) trabajando como desarrollador único (practicante), lo que limita el alcance a funcionalidades core y requiere priorización estricta. La app móvil Android (Kotlin + Firebase) queda como funcionalidad opcional para fases posteriores. El desarrollo individual puede extender los tiempos estimados comparado con un equipo.
2
Presupuesto Mínimo (Proyecto Académico)
Proyecto de practicante con presupuesto mínimo (aprox. USD 30), utilizando herramientas gratuitas/educativas, hardware propio y servicios cloud en tier gratuito. Requiere priorización de funcionalidades esenciales y uso eficiente de recursos gratuitos disponibles.
3
Integración con SUNAT
Dependencia de la disponibilidad y estabilidad de las APIs de SUNAT y del OSE seleccionado. Cualquier cambio en normativas o APIs de SUNAT puede requerir ajustes en el sistema.

2.10 Riesgos iniciales
Riesgo Descripción
1
Problemas de Integración con SUNAT
La integración con las APIs de SUNAT puede presentar dificultades técnicas o cambios en normativas que requieran ajustes en el sistema. Mitigación: Prototipo temprano, pruebas con ambiente de pruebas de SUNAT, diseño flexible.
2
Retrasos en Desarrollo por Complejidad de IA
La implementación de modelos de inteligencia artificial puede tomar más tiempo del estimado, especialmente el entrenamiento y optimización de modelos. Mitigación: Priorización de modelos básicos, integración incremental, uso de bibliotecas probadas.
3
Resistencia al Cambio de Usuarios
El dueño/empleado puede tener resistencia a adoptar el nuevo sistema, especialmente si requiere cambios significativos en sus procesos actuales. Mitigación: UI intuitiva, capacitación adecuada, demostración temprana de beneficios, soporte continuo.
